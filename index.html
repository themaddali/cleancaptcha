How to Use

Regular Custom

Registration

You are required to register only if you want to have custom questions.
Use Case: If you have a service related company, you can have a captcha related to service and questions based on your company.
Advantage: You get to promote and market along with solving the issue.
Details

Get Identification Code

Please save the unique identification code you get as you register. This code is required for you to send in the subsequent calls to idently yourselves. Without this code you end up getting captchas from public pool.
Details

Ask for a new Captcha (REST - POST)

Make an AJAX call from your website / page / blog directly to our server.
Public POST Call: http://www.cleancaptcha.com/CleanCaptcha/api/getCleanCaptcha
Custom POST Call: http://www.cleancaptcha.com/CleanCaptcha/api/getCleanCaptcha/?companyAuthCode=validauthcode
Response: { questionId: "qid002" question: "How many months in a year?" status: "Fail" }
Details

Collect User response (GUI - HTML - JS)

Have a HTML / JS mechanism to capture the response
HTML code: input type='text' id='CleanCaptch-UserInput'
Details

Validate the response (REST - POST)

Make an AJAX call from your website / page / blog directly to our server with the response from user.
POST Call: http://www.cleancaptcha.com/CleanCaptcha/api/validateResponse?questionId=questionid&answer=useranswer
Payload: { "questionId": "qid003", // Question ID from GET call. "answer": "8" // Response from User collected }
Response - Fail: { questionId: "qid002" // New Question ID question: "How many months in a year?" // New Question to Ask status: "Fail" }
Response - Success: { questionId: null question: null answer: null status: "Pass" // Cheers }
Details

Act by Response (GUI - HTML - JS)

Based on the response of Pass / Fail from previous call, just proceed what works best for your website.
